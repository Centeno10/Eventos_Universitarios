Opción 1: Thymeleaf (MVC)

Utiliza el patrón MVC tradicional con renderizado del lado del servidor usando Thymeleaf como motor de plantillas.

  • Renderizado en el servidor
  • Páginas HTML dinámicas
  • Formularios tradicionales
  • Navegación con recargas de página

Opción 2: HTML/JS + API REST

Utiliza HTML estático con JavaScript para consumir la API REST, siguiendo el patrón SPA (Single Page Application).

  • Renderizado en el cliente
  • Consumo de API REST
  • Actualización dinámica sin recargas
  • JavaScript con Fetch API
Ver Demo API REST

Información técnica

Endpoints disponibles:

  • Vista Thymeleaf: /eventos (Controlador MVC)
  • Vista HTML/JS: /static.html (Archivo estático)
  • API REST: /api/eventos (JSON)
  • Base de datos H2: /h2-console

Puerto del servidor: 8080
